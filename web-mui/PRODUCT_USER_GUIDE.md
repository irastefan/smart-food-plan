# SmartFoodPlan User Guide

Этот файл описывает продукт с точки зрения пользователя.

Его можно использовать в двух целях:
- как внутреннюю инструкцию для пользователей
- как основу для дизайна продающего одностраничного сайта

## 1. Что это за продукт

приложение для повседневного управления питанием, покупками, рецептами, уходом за собой и AI-помощником.

Продукт помогает пользователю:
- планировать еду по дням
- следить за калориями и макросами
- хранить рецепты и продукты
- быстро собирать список покупок
- вести трекинг веса и замеров тела
- планировать self-care routines по дням недели
- использовать AI-агента для добавления, изменения, анализа и организации данных

## 2. Для кого продукт

Подходит для пользователей, которые хотят:
- держать питание и быт в одной системе
- планировать рацион на каждый день
- видеть общую картину по калориям, БЖУ и прогрессу
- быстро добавлять продукты, рецепты и покупки
- использовать AI не как чат ради чата, а как рабочий инструмент
- объединить meal planning, shopping list, recipes и self-care в одном интерфейсе

## 3. Основные разделы

### Meal Plan

Главный раздел для планирования питания по дням.

Что умеет:
- отображать день с разбивкой на:
  - breakfast
  - lunch
  - dinner
  - snack
- добавлять продукты, рецепты и manual items
- использовать историю недавно добавленных элементов
- копировать приём пищи на другой день
- показывать total calories
- показывать macro balance
- визуально отмечать перебор по калориям и макросам
- анализировать день или отдельный приём пищи через AI
- добавлять продукты в shopping list

Особенности UX:
- выбор даты
- быстрые действия для каждого приёма пищи
- история последних добавленных элементов
- AI в контексте выбранного дня

### Body Metrics

Встроен в Meal Plan.

Что умеет:
- вести ежедневные измерения тела
- по умолчанию показывать только основные поля:
  - weight
  - bust
  - waist
  - hips
- настраивать видимые поля в Settings
- показывать график по выбранному параметру
- менять период отображения графика

Пользовательский смысл:
- приложение объединяет питание и телесный прогресс
- можно видеть не только еду, но и реальные изменения тела

### Recipes

Раздел для хранения и редактирования рецептов.

Что умеет:
- создавать рецепты вручную
- создавать рецепты через AI
- редактировать существующие рецепты
- открывать отдельную страницу рецепта
- смотреть ингредиенты, шаги и nutritional info
- добавлять ингредиенты рецепта в shopping list
- использовать AI для изменения рецепта в контексте текущей страницы

Пользовательский смысл:
- рецепты становятся рабочими объектами, а не просто заметками
- их можно использовать в meal plan и shopping list

### Products

База продуктов пользователя.

Что умеет:
- создавать и редактировать продукты
- открывать карточку продукта
- хранить nutrition per 100 g / serving
- добавлять продукт в shopping list
- использовать продукты дальше в meal plan и recipes

Пользовательский смысл:
- единый справочник продуктов
- меньше ручного ввода в ежедневных сценариях

### Shopping List

Раздел списка покупок.

Что умеет:
- группировать товары по категориям
- создавать категории
- добавлять элементы вручную
- добавлять товары из:
  - products
  - recipes
  - meal plan
  - self care
- отмечать товары как купленные
- удалять товары
- использовать AI для организации shopping list

Особенности UX:
- компактные элементы
- зачёркивание и выделение купленного товара
- item-level loading при изменении состояния
- общие toast-уведомления

### Self Care

Раздел для планирования рутин ухода за собой по дням недели.

Что умеет:
- показывать неделю как набор дней
- фильтровать по конкретному дню недели
- по умолчанию открывать сегодняшний день
- создавать slot внутри дня
- добавлять шаги внутрь slot
- редактировать и удалять slot и шаги
- копировать slot на несколько дней недели
- если slot с таким именем уже существует в выбранном дне, копировать шаги в существующий slot
- добавлять шаги self-care в shopping list
- использовать AI-агента в контексте выбранного дня

Пользовательский смысл:
- приложение покрывает не только еду, но и ежедневные routines
- можно планировать self-care так же наглядно, как meal plan

## 4. AI Agent

AI Agent — это общий рабочий компонент приложения, а не отдельный “демо-чат”.

Он открывается как popup/dialog и используется по всему продукту.

### Что умеет AI Agent

- работать в контексте текущей страницы
- понимать, где находится пользователь:
  - meal plan
  - recipes
  - shopping list
  - self care
- добавлять и изменять данные через tools
- задавать уточняющие вопросы, если данных недостаточно
- анализировать meal plan
- работать с shopping list
- помогать с рецептами
- использовать изображения
- использовать голосовой ввод

### Важная логика AI

- агент учитывает пользовательские дополнительные инструкции из Settings
- агент учитывает выбранный язык ответа отдельно от языка интерфейса
- агент не должен показывать internal IDs пользователю
- агент должен пытаться исправлять очевидные опечатки и ошибки speech-to-text
- если уверенности нет, агент должен уточнить, а не угадывать

### Как AI ощущается для пользователя

- AI показывает обычный typing state
- при выполнении tool виден отдельный execution state
- завершённые действия сохраняются в истории сообщений как отдельные статусы
- пользователь видит, что именно было:
  - added
  - removed
  - updated
  - copied
- после этого идёт финальный ответ ассистента

## 6. Мобильный опыт

Продукт адаптирован под mobile.

Что важно:
- нижняя навигация как app-like dock
- центральная кнопка AI Agent
- отдельный mobile menu через More
- popup AI не должен перекрывать нижнее меню
- сообщения и блоки оптимизированы для маленьких экранов

## 7. Мультиязычность и RTL

Приложение поддерживает:
- English
- Russian
- Hebrew

Что уже важно для пользователя:
- интерфейс может быть на одном языке, а AI отвечать на другом
- для Hebrew поддержан RTL
- неделя в self-care для Hebrew начинается с Sunday

## 8. Пользовательские сценарии

### Сценарий 1. Планирование дня

Пользователь:
- открывает Meal Plan
- выбирает день
- добавляет продукты и рецепты в breakfast/lunch/dinner/snack
- смотрит калории и макросы
- при необходимости анализирует день через AI

### Сценарий 2. Быстрое добавление покупок

Пользователь:
- добавляет товар в shopping list из recipe, product, meal plan или self-care
- получает автоматически заполненную категорию
- отмечает купленные товары

### Сценарий 3. Работа с рецептами

Пользователь:
- создаёт рецепт
- редактирует ингредиенты и шаги
- добавляет ингредиенты в shopping list
- использует AI, чтобы изменить рецепт

### Сценарий 4. Уход за собой на неделю

Пользователь:
- открывает Self Care
- выбирает нужный день
- добавляет slot
- добавляет шаги
- копирует slot на несколько дней
- использует AI для планирования дня

### Сценарий 5. Прогресс тела

Пользователь:
- записывает вес и замеры
- смотрит графики
- управляет видимостью полей

## 9. Что особенно ценно для пользователя

- всё в одном месте: еда, рецепты, покупки, self-care и AI
- AI встроен в реальные сценарии, а не вынесен в отдельный абстрактный чат
- единая логика добавления в shopping list из разных разделов
- наглядные калории, макросы и body metrics
- мобильный UX, похожий на приложение
- поддержка нескольких языков и RTL
